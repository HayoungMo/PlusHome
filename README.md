# PlusHome

가구 쇼핑과 인테리어 서비스를 연결하는 주거 통합 플랫폼

## 프로젝트 소개

PlusHome은 단순히 가구를 판매하는 쇼핑몰이 아니라, 가구 구매, 인테리어 상담, 사업자 운영 관리를 하나의 흐름으로 연결한 주거 통합 서비스 플랫폼입니다.

사용자는 가구를 구매하고 인테리어 서비스를 신청할 수 있으며, 사업자는 상품, 주문, 견적, 일정, 통계 등을 한 곳에서 관리할 수 있습니다.

실제로 가구를 구매하는 사용자는 공간 구성이나 인테리어에 대한 고민도 함께 하게 됩니다. 또 사업자는 상품 등록, 주문 확인, 배송 관리, 교환/반품 처리, 통계 확인 등 여러 업무를 동시에 처리해야 합니다.

그래서 PlusHome은 사용자 입장에서는 구매와 신청이 자연스럽게 이어지고, 사업자 입장에서는 운영 데이터를 한 곳에서 관리할 수 있는 구조를 목표로 했습니다.

## 프로젝트 목표

이 프로젝트의 목표는 단순한 기능 구현을 넘어, 풀스택 개발자로서 서비스 전체 흐름을 설계하고 구현하는 경험을 쌓는 것이었습니다.

- React에서 사용자와 사업자 화면 구현
- Spring Boot에서 API와 비즈니스 로직 구성
- MyBatis와 Oracle을 통한 데이터 관리
- 프론트엔드, 백엔드, 데이터베이스 흐름을 팀 단위로 연결
- 사용자 서비스와 사업자 운영 서비스를 하나의 플랫폼으로 통합

## 실행 방법

### Backend

```bash
./gradlew bootRun
```

Windows 환경에서는 다음 명령을 사용할 수 있습니다.

```bash
gradlew.bat bootRun
```

### Frontend

```bash
cd src/main/frontend
npm install
npm start
```

또는 yarn을 사용하는 경우:

```bash
cd src/main/frontend
yarn install
yarn start
```

## 환경 설정

백엔드와 데이터베이스 연결 정보는 `src/main/resources/application.properties`에서 관리합니다.

프론트엔드 개발 서버의 백엔드 프록시 설정은 다음 파일을 확인합니다.

- `src/main/frontend/src/setupProxy.js`
- `src/main/frontend/src/http-common.js`

Oracle DB 연결 정보, 백엔드 서버 주소, CORS 설정은 실행 환경에 맞게 조정해야 합니다.

## 프로젝트 의의

PlusHome은 가구 쇼핑, 인테리어 상담, 사업자 운영 관리라는 서로 다른 기능을 하나의 서비스 흐름으로 연결한 프로젝트입니다.

팀원들은 각자의 담당 기능을 구현하는 데 그치지 않고, 프론트엔드, 백엔드, 데이터베이스가 어떻게 연결되는지 함께 이해하며 전체 서비스 구조를 완성하는 것을 목표로 개발했습니다.
